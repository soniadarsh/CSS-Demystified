Block Vs Inline Elements

HTML (Hypertext Markup Language) elements historically were categorized as either "block-level" elements or "inline-level" elements. Since this is a presentational characteristic it is nowadays specified by CSS in the Flow Layout. A Block-level element occupies the entire horizontal space of its parent element (container), and vertical space equal to the height of its contents, thereby creating a "block". Browsers typically display the block-level element with a newline both before and after the element. You can visualize them as a stack of boxes. Note: A block-level element always starts on a new line and takes up the full width available (stretches out to the left and right as far as it can).

Usage

Block-level elements may appear only within a body element.

Block-level vs inline

There are a couple of key differences between block-level elements and inline elements:

Content model

Generally, block-level elements may contain inline elements and (sometimes) other block-level elements. Inherent in this structural distinction is the idea that block elements create "larger" structures than inline elements.

Default formatting

By default,block-level elements begin on new lines, but inline elements can start anywhere in a line. Inline elements are those which only occupy the space bounded by the tags defining the element, instead of breaking the flow of the content. Note: An inline element does not start on a new line and only takes up as much width as necessary.

Conceptual differences

In brief, here are the basic conceptual differences between inline and block-level elements: Content model Generally, inline elements may contain only data and other inline elements. You can't put block elements inside inline elements. Formatting By default, inline elements do not force a new line to begin in the document flow. Block elements, on the other hand, typically cause a line break to occur (although, as usual, this can be changed using CSS).

Block-level Elements

The following is a complete list of all HTML "Block-level Elements" elements (although "block-level" is not technically defined for elements that are new in HTML5).

1. "address": Contact information.
2. "article": Article content.
3. "dialog": Dialogbox.
4. "dd": Describes a term in a description list.
5. "div": Document division.
6. "dl": Description list.
7. "dt": Description list term.
8. "fieldset": Field set label.
9. "figcaption": Figure caption.
10. "figure": Groups media content with acaption (see "figcaption").
11. "footer": Section or page footer.
12. "form": Inputform.
13. "h1", "h2", "h3", "h4" "h5", "h6": Heading levels 1-6. "header" Section or page header.
14. "hgroup": Groups header information.
15. "hr": Horizontal rule (dividing line).
16. "li": List item.
17. "main": Contains the central content unique to this document.
18. "nav": Contains navigation links.
19. "ol": Ordered list.
20. "p": Paragraph.
21. "pre": Preformatted text.
22. "section": Section of a web page.
23. "table": Table.
24. "ul": Unordered lists.
